## Introduction

BooFi's codebase is set up in a monorepo via [Turborepo](https://turbo.build/repo) and is fully [open-source on GitHub](https://git.new/boofi).

Here's the monorepo structure:

```
apps
├── api
├── dashboard
├── docs
├── engine
├── mobile
├── website
packages
├── documents
├── email
├── events
├── import
├── inbox
├── jobs
├── kv
├── location
├── notification
├── supabase
├── tsconfig
├── ui
├── utils
```

The `apps` directory contains the code for:

- `api`: The backend for BooFi (supabase).
- `dashboard`: BooFi's application ([app.boofi.ai](https://app.boofi.ai)).
- `docs`: BooFi's documentation site ([boofi.ai/docs](https://boofi.ai/docs)).
- `engine`: BooFi's bank engine ([boofi.ai/engine](https://boofi.ai/engine)).
- `mobile`: BooFi's mobile app.
- `website`: BooFi's website app ([boofi.ai](https://boofi.ai)).

The `packages` directory contains the code for:

- `documents`: OCR for reciepts.
- `events`: Analytics library.
- `import`: CSV-importer.
- `inbox`: Inbox utils.
- `jobs`: Background jobs.
- `kv`: Key value storage client.
- `location`: Location related utils.
- `notification`: Notifications client.
- `supabase`: Queries and client.
- `tsconfig`: The TypeScript configuration for BooFi's web app.
- `ui`: UI component library.
- `utils`: A collection of utility functions and constants used across BooFi's codebase.

## Running BooFi locally

To run BooFi locally, you'll need to set up the following:

- A [Supabase](https://supabase.com/) account
- An [Upstash](https://upstash.com/) account

## Step 1: Local setup

First, you'll need to clone the BooFi repo and install the dependencies.

<Steps>

  <Step title="Clone the repo">

First, clone the [boofi.ai repo](https://d.to/github) into a public GitHub repository.

    ```bash Terminal
    git clone https://github.com/boofi-ai/boofi.git
    ```

  </Step>

  <Step title="Install dependencies">

    Run the following command to install the dependencies:

    ```bash Terminal
    bun i
    ```

  </Step>

</Steps>
